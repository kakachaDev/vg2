# Contributing to VG2

We welcome contributions! Please follow these guidelines.

## Development Setup

1. Clone the repository
2. Run `npm install` to install dependencies
3. Run `npm run build` to build all packages
4. Run `npm test` to run tests

## Code Style

- We use Prettier for formatting. Run `npm run format` before committing.
- Follow the existing TypeScript conventions.
- Use meaningful variable and function names.
- Write tests for new features (Vitest).

## Pull Request Process

1. Fork the repository and create a feature branch.
2. Make your changes, ensuring tests pass (`npm test`).
3. Update documentation if needed.
4. Submit a pull request with a clear description.

## Commit Messages

We follow [Conventional Commits](https://www.conventionalcommits.org/):
- `feat:` for new features
- `fix:` for bug fixes
- `docs:` for documentation
- `test:` for tests
- `refactor:` for code changes that neither fix bugs nor add features
- `chore:` for maintenance tasks

## Reporting Issues

Use GitHub Issues to report bugs or request features. Provide as much detail as possible.
