# Сервер для 2D игры — концепция для понимания проекта

## Что это вообще такое

Сервер, который держит в себе состояние игрового мира и разрешает клиентам в этом мире что-то делать. Клиент — просто терминал для отображения. Сервер решает всё.

Каждый игрок подключается к серверу по websocket (Socket.io), отправляет команды ("иду вправо", "бью моба"), сервер проверяет, можно ли, и если да — применяет изменения и рассылает обновления всем, кто рядом.

Если клиент попытается считерить — сервер просто проигнорирует или пошлет правильное состояние.

## Масштаб и инфраструктура

Один сервер должен выдерживать **~1000 одновременных игроков**.

Но миров может быть несколько:
- **Мир-хаб** — главный, где игроки тусуются, общаются, выбирают куда идти
- **Игровые миры (инстансы)** — отдельные серверы под конкретные активности (подземелья, ивенты, мини-игры)

Клиент может **переключаться между ними** — буквально отключается от одного сервера и подключается к другому. При этом данные персонажа (инвентарь, статы) где-то хранятся централизованно (например, в Redis или БД), чтобы при переходе всё подтянулось.

## Чем этот сервер отличается от обычного

В типичном мультиплеере клиент часто сам решает, где он находится, и просто сообщает остальным. Здесь всё наоборот:

1. Клиент шлет **намерение** ("хочу шагнуть")
2. Сервер проверяет (физика, коллизии, правила мира)
3. Сервер применяет **только если ок**
4. Сервер рассылает **новое положение всем**

Это называется **authoritative server** — сервер как источник правды. Никаких "клиент сам решил".

## Что делает сервер

- Держит в памяти всех игроков онлайн
- Держит чанки мира (тайлы, объекты)
- Получает действия от клиентов и валидирует их
- Обновляет мир
- Рассылает дельту (изменения) всем, кого это касается
- Отвечает за переходы между мирами (disconnect → connect к другому)

## Чего сервер НЕ делает

- Не рендерит графику
- Не хранит постоянно всю БД в памяти (только кеш)
- Не обрабатывает физику на каждый тик для всего мира сразу (только то, что рядом с игроками)
